# VERDICT Brand System

## Core Positioning

VERDICT is the machine court for agent economies.

It is not just a crypto app, a task board, or a reputation layer. It is the enforcement layer that turns agent commitments into economically real outcomes through deterministic onchain resolution.

### Positioning Statement

For agent economies that need real accountability, VERDICT is the deterministic enforcement layer that turns task commitments into onchain outcomes, because it resolves delivery through narrow, verifiable state instead of humans, oracles, or LLM judgment.

## Category

VERDICT should be presented as:

- the machine court for agent economies
- the enforcement layer for autonomous work
- deterministic onchain SLA enforcement

VERDICT should not be presented as:

- another Web3 productivity tool
- an AI agent marketplace
- a generic escrow app
- a speculative crypto protocol

## Brand Pillars

### 1. Determinism

VERDICT resolves from verifiable state, not opinion.

### 2. Consequence

VERDICT makes commitments costly to break.

### 3. Trust Through History

VERDICT lets delivery history compound into pricing power through VerdictScore.

### 4. Institutional Credibility

VERDICT should feel closer to infrastructure, law, and financial systems than to hype-driven crypto aesthetics.

## Voice and Tone

### Voice Attributes

- Precise
- Severe
- Credible
- Calm

### What That Means

Precise:
- says exactly what the system does
- does not exaggerate

Severe:
- treats commitments as serious
- avoids playful language in critical surfaces

Credible:
- sounds like infrastructure, not marketing theater
- prefers proof, numbers, and mechanisms over adjectives

Calm:
- does not yell
- does not use hype language to create urgency

### Avoid

- "revolutionary"
- "next-gen"
- "magic"
- "trustless vibes"
- "crypto-native disruption"

## Visual Direction

The identity should feel:

- institutional
- sharp
- metallic
- black-field dominant
- verdict-red accented

The new logo already supports this direction: it reads like a sharpened judicial V, a crest, and a machine-grade emblem at once.

## Color System

- Ink: `#050505`
- Paper: `#F5F5F2`
- Signal Red: `#E22B2B`
- Signal Red Bright: `#FF5A5A`
- Signal Red Deep: `#961616`
- Steel: `#6C6C6C`
- Muted Text: `#BDBDB7`

### Color Ratio

- 75% black / near-black
- 15% white / light neutral
- 10% red and steel accents combined

Red should behave like a ruling signal, not background wallpaper.

## Typography

- Display: `Syne`
- Body: `DM Sans`
- Technical / UI / system labels: `IBM Plex Mono`

### Typographic Behavior

- display lines should be compact and high-contrast
- body copy should stay clean and neutral
- mono should be used for chain, tx, score, and protocol details
